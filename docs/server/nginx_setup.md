# Nginx & Let's Encrypt Production Deployment Guide

This guide details how to configure a production-grade gateway VPS for `lfr-tunnel`. It covers setting up **Nginx** as a reverse proxy, configuring **Let's Encrypt** with wildcard SSL certificates (via the Cloudflare DNS-01 API challenge), enforcing **HTTP-to-HTTPS redirection**, and setting up a root-level redirect to `liferay.com`.

---

## 1. Prerequisites & Installation

To run this gateway, you will need:
*   A VPS running Debian/Ubuntu with a public IP.
*   DNS records resolving to your VPS (configured in DNS-only mode).
*   API credentials for your Cloudflare account to automate Let's Encrypt wildcard verification.

On your VPS, install Nginx, Certbot, and the Certbot Cloudflare DNS plugin:

```bash
sudo apt update
sudo apt install -y nginx certbot python3-certbot-nginx python3-certbot-dns-cloudflare
```

---

## 2. Obtain Let's Encrypt Wildcard Certificates

Because `lfr-tunnel` routes traffic based on wildcard subdomains (e.g. `*.lfr-demo.se`), we must obtain a wildcard SSL certificate. The standard HTTP-01 challenge cannot verify wildcards, so we must use the **DNS-01 challenge** via Cloudflare's API.

1.  **Create a Cloudflare API Token**:
    *   Go to your Cloudflare Dashboard -> My Profile -> API Tokens.
    *   Create a token with the template: **Edit zone DNS**.
    *   Limit the token's scope to your domains (`lfr-demo.se` and `lfr-demo.online`).

2.  **Save the Credentials on the VPS**:
    Create a secrets file at `/etc/letsencrypt/cloudflare.ini` (secure it so only root can read it):

    ```ini
    # /etc/letsencrypt/cloudflare.ini
    dns_cloudflare_api_token = YOUR_CLOUDFLARE_API_TOKEN
    ```

    Apply secure permissions:
    ```bash
    sudo chmod 600 /etc/letsencrypt/cloudflare.ini
    ```

3.  **Run Certbot to Fetch Certificates**:
    Fetch wildcard certificates for both domains.

    ```bash
    sudo certbot certonly \
      --dns-cloudflare \
      --dns-cloudflare-credentials /etc/letsencrypt/cloudflare.ini \
      --agree-tos \
      --no-eff-email \
      -m your-email@liferay.com \
      -d lfr-demo.se \
      -d *.lfr-demo.se \
      -d lfr-demo.online \
      -d *.lfr-demo.online
    ```

---

## 3. Configure Nginx

Create an Nginx server configuration file at `/etc/nginx/sites-available/lfr-tunnel`:

```nginx
# Map block to handle WebSocket Upgrade headers correctly
map $http_upgrade $connection_upgrade {
    default upgrade;
    ''      close;
}

# ==============================================================================
# 1. HTTP to HTTPS Force Redirect
# ==============================================================================
server {
    listen 80;
    listen [::]:80;
    server_name lfr-demo.se *.lfr-demo.se lfr-demo.online *.lfr-demo.online;

    return 301 https://$host$request_uri;
}

# ==============================================================================
# 2. Gateway Control Plane & Root Redirect to Liferay.com
# ==============================================================================
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name lfr-demo.se tunnel.lfr-demo.se lfr-demo.online tunnel.lfr-demo.online;

    ssl_certificate /etc/letsencrypt/live/lfr-demo.se/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/lfr-demo.se/privkey.pem;

    # Secure SSL Protocols and Ciphers (generated by Certbot)
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    # Root landing/holding page redirects directly to Liferay corporate page
    location / {
        return 307 https://www.liferay.com;
    }

    # Proxy CLI registration endpoints to lfr-tunneld running on port 8080
    location /api/register {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Proxy Chisel control channel WebSockets
    location /tunnel {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# ==============================================================================
# 3. Wildcard Subdomain Data Plane Routing
# ==============================================================================
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name *.lfr-demo.se *.lfr-demo.online;

    ssl_certificate /etc/letsencrypt/live/lfr-demo.se/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/lfr-demo.se/privkey.pem;

    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;

        # WebSocket Tunnel headers
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;

        # Standard Forwarding Headers (essential for Liferay instances)
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Proto https;

        # Increase request size and timeouts for deploying large Liferay bundles
        client_max_body_size 500M;
        proxy_connect_timeout 120s;
        proxy_send_timeout 120s;
        proxy_read_timeout 120s;
    }
}
```

Enable the site configuration and restart Nginx:
```bash
sudo ln -s /etc/nginx/sites-available/lfr-tunnel /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

---

## 4. Run `lfr-tunneld` Gateway Daemon

With Nginx handling the public interface, `lfr-tunneld` is configured to bind to **localhost** (`127.0.0.1:8080`) instead of running directly on port 443.

Update your `server-config.yaml` on the VPS:

```yaml
domain1: "lfr-demo.se"
domain2: "lfr-demo.online"
bind_addr: "127.0.0.1:8080"
chisel_bind_addr: "127.0.0.1:8081"
auth_token: "YOUR_SHARED_SECRET_TOKEN"
# ssl_cert_file and ssl_key_file are omitted since Nginx terminates SSL
```
